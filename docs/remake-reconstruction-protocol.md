# Remake Reconstruction Protocol

本协议用于约束“用户给一个片段、短片或链接，要做复刻和原型重构”的完整前置流程。目标是先拿到参考片 DNA，再根据用户选择决定复刻哪些层，不直接跳到资产、故事板或 Seedance 提示词。

## 1. Requirement Intake 必问项

在生成任何资产、故事板或 Seedance 提示词前，必须先向用户收集复刻目标。用户没有明确选择时，默认只进入分析，不进入生产。

### 1.1 复刻范围

让用户从以下选项中选择一个或多个：

| Option | Meaning | Output Impact |
|---|---|---|
| `shot_rhythm` | 复刻镜头数量、切点、景别推进和剪辑节奏 | 故事板按原片镜头数设计，Seedance 按原片切点写 |
| `camera_language` | 复刻推拉摇移跟、角度、高低机位、镜头速度曲线 | 每镜写清镜头运动和镜头衔接 |
| `story_structure` | 复刻剧情功能、信息释放、转折和结尾爆点 | 生成新剧情映射，不复制原剧情表达 |
| `character_method` | 复刻角色出场方式、表演方法、动作语言 | 资产板锁定新角色，保留出场方法 |
| `scene_method` | 复刻空间调度、场景压力、前中后景关系 | 场景板锁定新空间地理和机位 |
| `style_look` | 复刻光影、色彩、材质、画面质感 | 风格锁写入资产、故事板和视频提示词 |
| `dialogue_function` | 复刻台词/旁白功能和信息节奏 | 只保留功能，生成新台词方向，不复制原句 |
| `sound_sfx` | 复刻声音功能、音效点、静默、冲击点 | 输出原创音效清单，不复用原声音素材 |
| `vfx_method` | 复刻特效触发、形态、节奏和画面功能 | 生成新 VFX 规则，不复制原特效造型 |
| `full_method_remake` | 以上全部方法层都分析和迁移 | 输出完整 DNA、资产、故事板、Seedance 包 |

### 1.2 变化范围

继续确认用户想替换哪些变量：

- 新主题 / 新剧情
- 新角色身份
- 新场景
- 新道具 / 产品 / 法术系统
- 新台词 / 新旁白
- 新风格
- 新画幅
- 新时长
- 新平台

### 1.3 用户自带资产

如果用户同时提供参考片和自己的角色图、场景图、道具图、产品图或素材包，必须进入 `REFERENCE_PLUS_USER_ASSETS` 分支。

必须先确认：

- 用户提供了哪些资产：角色、场景、道具、产品、logo、字体、声音或其他。
- 哪些资产必须一比一保留，哪些资产允许保守补全。
- 用户资产的媒介 DNA：真人摄影、动漫/二次元、CG/3D、插画、产品摄影、真实场景、混合媒介。
- 用户希望用自带资产替换参考片中的哪一类资产：原主角、原配角、原场景、原道具、原产品、原特效核心物。
- 缺失视角：正面、侧面、背面、表情、手部、材质、尺度、场景地图、道具持握、状态变化。

处理原则：

- 用户自带资产是主资产来源，不是灵感图。
- 参考片资产只提供“镜头功能位置、出场方法、构图关系、动作节奏、运镜方法”，不得覆盖用户资产身份。
- 资产板必须按照用户提供的资产补全，而不是重新设计一套无关角色、场景或道具。
- 已有角色图要先做 `Visible / Inferred / Missing` 拆分；可见项一比一保留，缺失项只做保守补全。
- 已有场景图要补成九视角场景板和 MAP，保持原空间地理，不随参考片场景漂移。
- 已有道具图要补成多视角道具板，锁定形状、材质、比例、手部使用和状态变化。
- 如果用户资产与参考片方法冲突，优先保护用户资产身份，再调整镜头调度。

### 1.4 保留边界

必须提醒并确认：

- 可以保留镜头方法、节奏、情绪曲线、光影动机、构图逻辑。
- 不复制原人物脸、品牌、字幕、台词、音乐、logo、IP、可识别场景、原特效造型和原镜头画面。
- 如果用户要求“完全一样”，系统必须改写成“方法复刻 + 表达替换”。

## 2. DNA Deconstruction 必拆维度

前期分析必须覆盖以下维度，不能只写“高级、电影感、节奏好”。

| Dimension | Required Questions |
|---|---|
| 剧情 | 每个镜头改变了观众什么认知？开场、发展、转折、爆点分别是什么？ |
| 分镜 | 原片有多少镜头？每镜起止时间、景别、角度、构图中心是什么？ |
| 运镜 | 镜头是推、拉、摇、移、跟、升降、旋转、锁定还是手持？速度如何变化？ |
| 台词/旁白 | 台词或字幕承担什么功能？设定、反转、情绪、口号、解释还是节奏标记？ |
| 风格 | 光影、色彩、材质、画幅、镜头质感、人物渲染和后期质感是什么？ |
| 构图 | 主体在哪个安全区？前中后景如何分层？视觉引导线是什么？ |
| 音效 | 哪些声音点推动动作？哪些是氛围、冲击、静默、转场或终帧余音？ |
| 特效 | 特效从哪里触发？形态如何变化？服务动作、信息还是情绪？ |
| 剪辑 | 切点来自动作、视线、台词、声音、情绪还是视觉突变？ |
| 节奏 | 镜头时长分布如何？哪里快切、哪里停顿、哪里锁帧？ |

### 2.1 DNA Forensic Audit Gate

逐镜拆解之后、进入变量/不变量之前，必须执行 [DNA Forensic Audit System](dna-forensic-audit-system.md)。

15 秒以内片段必须逐帧/逐变化拆解，至少输出：

- `Frame Ledger`：逐帧或逐变化帧观察表。
- `Shot Ledger`：最终镜头表。
- `DNA Invariant Ledger`：不变量 DNA 表。
- `Omission Audit`：漏项审计。
- `DNA_SIGNOFF`：是否允许进入生产。

漏项审计必须显式检查：

- 主角是否有变身、换装、法身前态、能级升级。
- 道具是否有触发机制和状态变化。
- 法术/VFX 是否有触发、扩散、回拉、升级和终帧前态。
- 群体是否有被勒停、冻结、击退、后仰、跪下或散开。
- 终帧是否由前面动作因果推出来，还是突然出现。

如果 `DNA_SIGNOFF` 不是 `PASS`，不得进入资产、故事板或 Seedance。

## 3. Variable / Invariant Map

分析完成后必须输出变量和不变量。

### 3.1 不变量

不变量是可以学习的方法：

- 镜头数量和切点节奏
- 景别推进
- 运镜路径和速度曲线
- 构图逻辑
- 动作因果
- 剪辑逻辑
- 情绪曲线
- 声音功能
- 特效触发逻辑
- 信息释放顺序

### 3.2 变量

变量是必须替换的表达：

- 人物身份和脸
- 服装、妆容、发型
- 台词、字幕、片名、文案
- 品牌、logo、包装
- 原音乐和原音效素材
- 场景具体造型
- 道具和法术图形
- 特效具体形状
- 可识别构图和终帧画面

### 3.3 用户资产绑定

如果用户提供了自己的资产，变量表必须多一列 `user_asset_binding`：

| Reference Asset Function | User Asset Binding | Keep From User Asset | Learn From Reference |
|---|---|---|---|
| 原片主角承担操控者功能 | 用户角色 A01 | 脸、体型、服装、媒介风格、身份气质 | 出场顺序、景别推进、动作位置 |
| 原片战场承担空间压迫功能 | 用户场景 A02 | 空间地理、建筑轮廓、色彩基调 | 高位压入、前中后景压力、切点节奏 |
| 原片红线承担控制道具功能 | 用户道具 A03 | 道具形状、材质、比例、使用规则 | 触发节奏、手部特写、VFX 功能 |

没有用户资产绑定的项目，才可以重新设计新资产。

## 4. Storyboard Rules

故事板必须根据参考片的真实镜头数量和新剧情来设计，不得强行套固定 4 格。

### 4.1 Shot Count First

如果参考片是 12 秒，有 8 个镜头，就必须设计 8 个故事板格子。每格对应一个参考镜头的功能和新片镜头。  
如果参考片超过 10 个镜头，必须拆成多个故事板：

- `S01`: SH01-SH10
- `S02`: SH11-SH20
- 以此类推

### 4.2 Layout Rule

| Shot Count | Storyboard Layout |
|---:|---|
| 1-4 | 1x4 or 4x1, unused panels marked empty only if needed |
| 5-6 | 3x2 |
| 7-8 | 4x2 |
| 9-10 | 5x2 |
| >10 | split into multiple boards, max 10 shots per board |

### 4.3 Panel Beat Rule

每格必须写：

```text
SH## / timecode / shot size / camera move / action beat / active assets
```

每格必须对应当下剧情和当下节奏，不允许重复摆拍、空镜、同姿势复制。

## 4.5 Creative Direction Gate

在进入资产、故事板和 Seedance 生产前，必须给用户至少 3 个新片重构方向供选择。每个方向必须说明：

- 方向名
- 新主题
- 新主角/核心角色
- 新场景
- 新道具、产品或法术系统
- 保留的参考片 DNA
- 主要变化点
- 风格关键词
- 合规或生成风险

如果用户已经指定唯一方向，可以只围绕该方向继续；否则不得只给一个默认方向。

## 4.6 User Asset Board Gate

进入故事板和 Seedance 前，如果用户提供了资产，必须先完成用户资产板：

- `A01 / USER_CHAR_*`：用户角色资产板，基于用户图补全 M01-M08。
- `A02 / USER_SCENE_*`：用户场景资产板，基于用户图补全 V01-V09 和 MAP。
- `A03 / USER_PROP_*`：用户道具资产板，基于用户图补全 P01-P08。

资产板输出必须写清：

- Source：来自用户上传/提供的哪张图或素材。
- Visible：原图可见且必须保留的内容。
- Inferred：根据原图可以保守推断的内容。
- Missing：原图缺失、需要补全或需要用户确认的内容。
- Lock：不可改变项。
- Remake Function：它在复刻参考片里替代原片哪个资产功能。
- Later Seedance Reference：后续如何引用到 `A01/M03`、`A02/V04`、`A03/P06` 等模块。

## 5. Seedance 2.0 Reference Rules

Seedance 提示词必须明确写清“参考哪张资产图、锁定哪个标签、干净控制帧或文字故事板如何转视频”。带箭头、框线、标签、编号或说明文字的标注故事板只能给人审片，不得作为 Seedance 图片输入。

### 5.0 No-Asset Direct Rule

如果用户只提供参考片段、短片或链接，没有提供资产，也没有要求先出资产或故事板，只要求“直接给我一条 Seedance 2.0 提示词”，系统进入 `NO_ASSET_TEXT_ONLY_SEEDANCE` 分支。

处理方式：

- 仍然必须先完成 DNA 分析和 `DNA_SIGNOFF: PASS`。
- 不输出资产板，除非用户后续要求。
- 不输出故事板，除非用户后续要求。
- 最终只交付一条可复制的 Seedance 2.0 提示词和必要的验证清单。
- 不写不存在的资产编号、模块标签或 `@图片` 引用。
- 用短而稳定的文本锚点锁定新片主角、场景、道具/法术系统和风格。

局限必须明确：

- 角色一致性弱于资产支线。
- 道具形状、服装和场景稳定性弱于资产支线。
- 镜头切点控制弱于黑白故事板支线。
- 如果用户后续要求稳定角色或严格切镜，应升级到 A 线或 B 线。

### 5.1 Asset Reference Syntax

必须使用如下写法：

```text
参考 A01 / M03 FACE CLOSE-UP 锁定角色脸部。
参考 A01 / M05 ACTION HAND PROP LOGIC 锁定手部动作。
参考 A02 / V04 CAMERA A 锁定场景机位。
参考 A02 / MAP CHAR_ZONE / PROP_ANCHOR 锁定空间关系。
参考 A03 / P06 HAND LOGIC 锁定道具持握。
参考 A03 / P07 STATE 锁定道具状态。
```

不要只写“参考角色图”“参考场景图”“参考故事板”，必须写到图号和标签。

### 5.2 Storyboard Reference Syntax

必须使用如下措辞：

```text
将提供的 S01_CLEAN_CONTROL_FRAMES / [COLUMNS]x[ROWS] 干净控制帧制作成一段流畅的电影级动画视频。
严格保持 SH01-SH08 的镜头顺序、切点节奏、构图重心、动作方向和镜头连贯性。
不要上传或渲染带箭头、框线、绿色标签、编号、标题栏、文字标注或面板布局的 S01_ANNOTATED_STORYBOARD。
```

如果只有标注故事板，必须写成：

```text
S01_ANNOTATED_STORYBOARD 仅供导演理解，不作为 Seedance 图片输入。请根据 SH01-SH08 文字 timing、镜头运动、动作方向和资产引用生成连续视频。
```

如果有多个故事板：

```text
先使用 S01 完成 SH01-SH10，再使用 S02 完成 SH11-SH20。保持两个故事板之间的动作连续和空间连续。
```

### 5.3 Seedance Shot Block

每个 Seedance 段落必须包含：

- Reference Logic
- Asset Reference Map
- Storyboard Reference
- Shot-by-Shot Timing
- Motion Continuity
- Action / Blocking
- Camera Movement
- Lighting / VFX
- Sound Effects For Post
- Do Not Copy
- Negative Prompt

## 6. Output Gate

进入生产前必须确认：

- 用户已选择复刻范围。
- 已完成多维 DNA 拆解。
- 已输出变量 / 不变量表。
- 已确定参考片镜头数量和故事板数量。
- 故事板格数等于参考镜头数，或超过 10 镜头时已拆板。
- Seedance 引用写到资产图号和模块标签。
- BGM、台词、字幕、logo、IP、人物脸等风险已处理。
