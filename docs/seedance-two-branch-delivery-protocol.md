# Seedance 2.0 三支线交付协议

本协议用于复刻/重构项目进入后期视频生成阶段时选择交付路径。核心原则：参考片只在后台拆解阶段存在；最终模型提示词只描述新片本身。

## 1. 三条交付支线

### A 线：资产直出 Seedance

用途：身份一致性优先，用户指定角色/场景/道具必须严格生效时，默认使用 A 线。

输入：

- `@图片1`：角色身份资产，唯一锁定脸、发型、体型、服装、气质、手部。
- `@图片2`：场景资产，锁定空间布局、主光方向、前中后景、关键锚点。
- `@图片3`：道具/法术媒介资产，锁定形状、材质、尺度、持握/触发关系。
- 可选 `@图片4-@图片5`：只在确实必要时加入，比如手部机制或终帧法相资产。

提示词写法：

- 先写身份与资产绑定。
- 再写 12 秒片内切镜时间轴。
- 每个镜头只写一个主动作、一个主运镜、一个可见结果。
- 不上传故事板，不让任何临时视觉参考抢角色身份。

适用情况：

- 用户说“指定角色没生效”。
- 角色脸、服装、道具形状比镜头复刻更重要。
- 故事板支线连续两次污染身份或画面。
- 项目要先拿到可用成片，再追求更精确的切镜。

### B 线：黑白故事板参考 Seedance

用途：镜头顺序、切点节奏、构图重心、动作方向特别重要，并且已有合格黑白故事板时使用。

输入：

- `@图片1-A0N`：角色、场景、道具资产。
- `@图片X`：`S##_CLEAN_BW_STORYBOARD / [列]x[行]` 黑白手稿故事板。

硬条件：

- 故事板必须是黑白手稿草图。
- 不得有颜色、精修脸、精修服装材质、箭头、框线、标签、编号、文字、标题栏、白底 UI。
- 故事板必须由最终资产生成，不能出现临时角色脸、临时服装、临时发型或新场景身份。
- Seedance 提示词必须写明：资产图锁身份，故事板只锁运动、构图、切点和动作方向。

失败判定：

- 成片角色不是指定角色。
- 成片复制故事板临时人物。
- 出现故事板 UI、箭头、编号、文字、面板边框。
- 场景、道具或法术媒介被故事板替换。

任一失败，B 线回退到 A 线，不继续叠加更多提示词。

### C 线：无资产文本直出 Seedance

用途：用户只给参考片段、短片或链接，没有提供资产，也没有要求先出资产/故事板，只要求“直接给我一条 Seedance 2.0 提示词”时使用。

输入：

- 不上传资产图。
- 不上传故事板图。
- 不写不存在的资产编号或模块标签。
- 参考片只在后台 DNA 分析阶段使用，最终提示词只描述新片本身。

硬条件：

- 必须先完成 DNA 分析和 `DNA_SIGNOFF: PASS`。
- 必须确认复刻/重构范围；如果用户只说“直接复刻/重构”，默认按 `full_method_remake` 输出。
- 15 秒以内必须按真实镜头数写成一条连续视频提示词，例如 12 秒 8 镜就写 `SH01-SH08`。
- 不生成资产编号，不写 `参考 A01`、`参考 A02`，因为没有资产输入。

提示词写法：

- 先写新片的角色、场景、道具/法术系统文本设定。
- 再写逐镜 timing、景别、运镜、动作因果、台词口型、VFX、SFX。
- 每个镜头只承担一个主任务，避免模型把文字理解成散乱元素清单。
- 角色一致性只能靠文本锚点维持，所以角色锚点要短、稳定、反复出现在关键镜头。

适用情况：

- 用户想快速测试方向，不想先出资产。
- 用户没有准备角色、场景、道具图。
- 用户明确说“先别做资产，直接给 Seedance 提示词”。
- 项目处于创意验证，不追求高稳定角色身份。

风险：

- 角色脸、服装、道具和场景一致性弱于 A 线。
- 镜头切点控制弱于 B 线。
- 复杂变身、口型、道具机制和终帧法相可能需要多轮修正。
- 如果用户后续要求角色稳定或镜头严格，必须升级到 A 线或 B 线。

## 2. Prompt 精准原则

后台可以长分析，模型提示词必须短而清楚。

有效信息优先级：

1. 资产身份绑定。
2. 镜头时间轴和动作链。
3. 单一主运镜。
4. 空间和 VFX 因果。
5. 必要 negative。

避免：

- 把后台分析过程写进提示词。
- 把参考片、原片、学习、不要复制原片写进提示词。
- 一个镜头里塞多个运镜。
- 多个视觉参考同时定义同一个角色身份。
- 靠长篇 negative 试图修正互相冲突的正向输入。

## 3. A 线模板

```text
[VIDEO TASK]
生成一条完整连续的 [时长] 秒电影级视频，[画幅]，[fps]。视频包含 SH01-SH[N] 的片内切镜；这些镜头是同一条视频里的连续剪辑节奏，不要拆成多条视频，不要一镜到底。

[ASSET LOCK]
@图片1 / A01 是唯一角色身份源：锁定脸、发型、体型、服装、手部、气质。
@图片2 / A02 锁定场景空间：空间布局、光线方向、关键锚点。
@图片3 / A03 锁定道具/法术媒介：形状、材质、尺度、触发关系。
如果其他描述与资产冲突，以资产图为准。

[SHOT TIMING]
SH01 / 0.0-1.2s / [景别] / [一个主运镜] / [一个主动作 + 可见结果]
SH02 / ...

[CONTINUITY]
说明每个镜头如何通过动作、视线、线条方向、空间锚点或声音点接到下一个镜头。

[VOICE / SFX]
后期配音：[台词节奏，不生成字幕]。
无 BGM。SFX：[每镜 1-2 个音效点]。

[NEGATIVE]
字幕，画面文字，水印，logo，箭头，标签，故事板 UI，角色脸漂移，服装漂移，道具变形，场景漂移，多指，坏手，不可能运镜，线条杂乱遮挡主体。
```

## 6. 默认选择

- 如果用户没有资产、也没有要求资产/故事板，只要求一条 Seedance 提示词，默认交付 C 线。
- 如果用户提供或要求资产，默认先交付 A 线。
- 只有当用户明确要严格按故事板控制切镜，且黑白故事板通过身份审核时，才交付 B 线。
- 三条线可以同时交付，但必须明确标注推荐先跑哪条。
